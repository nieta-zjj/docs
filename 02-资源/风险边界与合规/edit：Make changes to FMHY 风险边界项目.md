## 摘要

fmhy/edit 是一个社区驱动、持续更新的免费互联网资源整理项目，覆盖隐私、AI、音视频、游戏、阅读、下载等多个主题。该项目在“资源聚合”之外，还提供内容准入规则、风险清单和安全过滤工具，用于降低不可信来源带来的风险。作为高风险内容生态下的索引仓库，它更适合用于信息检索与边界识别，而不是作为安全或合规保证。

## 功能与定位

- 面向公开网络资源提供主题化导航，定位为社区协作维护的资源索引仓库。
- 通过分类页面、月度更新帖和持续提交保持内容可读性与时效性。
- 同时建设风险治理能力，包括不可信来源提示与安全辅助工具入口。

## 典型使用场景

- 快速浏览某一主题下的可选工具与站点，并做初步对比。
- 在访问外部资源前，先参考仓库给出的风险提醒与不可信来源清单。
- 团队或个人在内部评估时，把它作为“候选来源发现池”，再做二次验证。
- 需要可控访问链路时，结合其镜像与自托管说明构建自用副本。

## 特色与差异点

- 社区驱动明显：通过 GitHub 与 Discord 持续吸收反馈并滚动修订。
- 内容边界明确：贡献规则中明确拒绝部分高风险或不符合定位的提交类型。
- 风险治理可见：专门维护不可信来源清单，并配套 SafeGuard/过滤列表思路。
- 可用性冗余：主站、镜像、备份与源码仓库并行，降低单点不可用影响。

## 使用方式概览

- 在线使用：直接通过主站或镜像按主题检索资源。
- 协作更新：通过 Issue、Pull Request、社区讨论提交修正建议。
- 自主部署：仓库提供文档站与 API 的部署说明，可用于自托管场景。

## 限制与注意事项

- 该项目涉及高风险内容生态，外部链接质量与安全性仍需使用者自行核验。
- 仓库中的“风险提示”属于社区维护信息，不等同于法律、合规或安全认证。
- 本仓库仅记录项目定位、结构与边界提示，不复述可操作的绕过授权或不当使用细节。
