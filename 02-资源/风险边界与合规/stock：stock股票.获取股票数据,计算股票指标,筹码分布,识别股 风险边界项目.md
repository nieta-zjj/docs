## 摘要

InStock（myhhub/stock）是一个面向量化分析场景的开源股票数据与选股系统，覆盖数据抓取、技术指标计算、K 线形态识别、策略回测与 Web 展示等环节。项目强调多维度数据整合与批量处理能力，并提供自动交易相关能力；但该方向同时涉及真实资金、数据源访问策略与合规边界，使用前应先完成风控与合法性评估。

## 功能与定位

- 面向 A 股与 ETF 的日常数据处理，覆盖行情、资金流向等多类信息。
- 提供指标计算、筹码分布、K 线形态识别与综合选股能力。
- 支持策略回测与结果可视化，定位为量化研究与选股辅助工具。
- 项目采用 Python 技术栈，仓库声明为 Apache License 2.0。

## 典型使用场景

- 研究型团队构建股票数据看板与指标分析流程。
- 对候选股票进行规则筛选后再执行回测验证。
- 在自建环境中串联数据抓取、计算、分析与展示流程。

## 特色与差异点

- README 声明覆盖 2200+ 维度信息，并提供多类选股维度组合能力。
- 同时覆盖指标、形态、筹码与策略模块，功能链路完整。
- 提供 Docker 镜像与 Web 界面，便于部署与跨设备查看。
- 仓库将自动交易作为可选模块，并对资金相关风险给出提醒。

## 使用方式概览

- 可按仓库文档在本地或容器环境部署，再执行数据处理与分析流程。
- 常见流程是先采集与更新数据，再进行指标/策略计算和回测，最后在 Web 端查看结果。
- 自动交易能力应与研究流程隔离，优先在低风险环境验证后再决定是否启用。

## 限制与注意事项

- 项目暂无正式 Release 版本，版本稳定性需结合提交历史自行评估。
- 自动交易涉及真实资金，任何策略结论都不应直接替代投资决策。
- 数据源访问可能涉及频率限制、服务条款与地区监管要求，需自行确认合法合规性。
- 本仓库仅记录项目定位、能力边界与风险提示，不复述可执行操作细节。
