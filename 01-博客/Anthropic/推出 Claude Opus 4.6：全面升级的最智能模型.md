# 推出 Claude Opus 4.6：全面升级的最智能模型

## 文档信息
- 来源：https://www.anthropic.com/news/claude-opus-4-6

## 摘要
**1) 一句话总结**
Anthropic 推出了全面升级的 Claude Opus 4.6 模型，在编程、代理任务和复杂推理上达到行业领先水平，并首次在 Opus 级别提供 100 万 Token 上下文窗口及自适应思考功能。

**2) 关键要点**
*   **基准测试表现卓越**：在 Terminal-Bench 2.0（代理编程）、Humanity’s Last Exam（复杂推理）和 GDPval-AA（高经济价值知识工作）等多项评估中达到行业领先水平（SOTA）。
*   **克服“上下文衰退”**：Beta 1 版本提供 100 万 Token 上下文窗口，在 100 万 Token 的“大海捞针”测试中得分高达 76%（前代 Sonnet 4.5 仅为 18.5%）。
*   **API 新增思考控制**：引入“自适应思考”功能，模型可自主决定深度推理时机；开发者可通过四个级别（低、中、高、最大）手动控制思考力度（Effort）。
*   **上下文压缩与大输出**：推出 Beta 版“上下文压缩”功能以自动总结旧上下文，支持更长期的代理任务；同时支持高达 12.8 万 Token 的单次输出。
*   **产品端生态扩展**：推出 Claude Code 代理团队（支持多代理并行协调），并大幅提升了 Claude 在 Excel（数据处理）和 PowerPoint（可视化与排版）中的应用能力。
*   **安全性提升**：保持极低的不良行为发生率，同时实现了近期所有 Claude 模型中最低的“过度拒绝率”（未能回答无害查询的情况）。
*   **定价与部署**：常规 API 定价保持不变（输入 $5/输出 $25 每百万 Token）；超过 20 万 Token 的长上下文适用高级定价；新增价格为 1.1 倍的“仅限美国推理”选项。

**3) 风险与不足**
*   **简单任务的成本与延迟风险**：模型在处理简单问题时可能会“过度思考”，从而增加不必要的成本和延迟（官方建议通过调低思考力度参数来解决）。
*   **网络安全滥用风险**：随着模型网络安全能力的增强，存在潜在的滥用风险；官方已开发 6 种新探针进行追踪，并计划在未来实施实时干预以阻止滥用。

## 正文
我们正在对旗下最智能的模型进行全面升级。

全新的 Claude Opus 4.6 在编程技能上超越了前代产品。它规划更周密，能更持久地执行代理任务，在大型代码库中运行更可靠，并具备更好的代码审查和调试技能来纠正自身错误。作为 Opus 级别模型的首创，Opus 4.6 在 Beta 1 版本中提供了高达 100 万（1M）Token 的上下文窗口。

Opus 4.6 还能将其提升的能力应用于各种日常工作：运行财务分析、开展研究，以及使用和创建文档、电子表格与演示文稿。在支持 Claude 自主多任务处理的 Cowork 环境中，Opus 4.6 可以代您将这些技能投入实际应用。

### 卓越的基准测试表现

该模型在多项评估中达到了行业领先水平（SOTA）。例如：
*   在代理编程评估 Terminal-Bench 2.0 中获得最高分。
*   在复杂的多学科推理测试 Humanity’s Last Exam 中领先所有其他前沿模型。
*   在 GDPval-AA（评估金融、法律等领域高经济价值知识工作表现的测试）中，Opus 4.6 领先行业次优模型（OpenAI 的 GPT-5.2）约 144 Elo 积分，领先其前代（Claude Opus 4.5）190 分。
*   在衡量在线定位难寻信息能力的 BrowseComp 测试中，表现优于其他任何模型。

**克服“上下文衰退”**
AI 模型常见的一个问题是“上下文衰退（Context rot）”，即当对话超过一定数量的 Token 时，性能会下降。Opus 4.6 的表现明显优于前代：在 100 万 Token 的 8 针 MRCR v2（一项测试模型在海量文本中检索“隐藏”信息能力的“大海捞针”基准测试）中，Opus 4.6 得分高达 76%，而 Sonnet 4.5 仅为 18.5%。这标志着模型在保持峰值性能的同时，其实际可用上下文容量实现了质的飞跃。

总体而言，Opus 4.6 在长上下文中查找信息的能力更强，在吸收信息后的推理能力更出色，整体专家级推理能力也有了大幅提升。

### 初步体验与实际应用

我们用 Claude 来构建 Claude。我们的工程师每天都使用 Claude Code 编写代码，每个新模型都会首先在我们自己的工作中进行测试。

我们发现，Opus 4.6 能够在无需提示的情况下，将注意力集中在任务最困难的部分，快速处理简单的部分，以更好的判断力处理模糊问题，并在长时间的会话中保持高效。

Opus 4.6 通常会进行更深入的思考，并在给出答案前仔细复核其推理过程。这在处理难题时能产生更好的结果，但在简单问题上可能会增加成本和延迟。如果您发现模型在特定任务上“过度思考”，我们建议通过 `/effort` 参数，将其默认的思考力度（高）调低至“中”。

### 产品与 API 更新

我们对 Claude、Claude Code 和 Claude 开发者平台进行了大幅更新，以充分发挥 Opus 4.6 的性能。

**Claude 开发者平台 (API) 新特性：**
*   **自适应思考 (Adaptive thinking)：** 过去，开发者只能在开启或关闭扩展思考之间二选一。现在，Claude 可以自行决定何时需要深度推理。在默认力度（高）下，模型会在有用时使用扩展思考，开发者也可调整力度级别使其更具选择性。
*   **思考力度控制 (Effort)：** 现在有四个级别可供选择：低、中、高（默认）和最大（max）。
*   **上下文压缩 (Context compaction，Beta 版)：** 长对话和代理任务经常会触及上下文窗口上限。上下文压缩会在对话接近设定阈值时自动总结并替换旧上下文，让 Claude 能够执行更长时间的任务而不受限制。
*   **100 万 Token 上下文 (Beta 版)：** 这是我们首个具备 1M 上下文的 Opus 级别模型。超过 20 万 Token 的提示词适用高级定价（输入/输出每百万 Token $10/$37.50），仅在 Claude 开发者平台提供。
*   **12.8 万输出 Token：** Opus 4.6 支持高达 128k Token 的输出，允许 Claude 完成大输出任务而无需拆分为多个请求。
*   **仅限美国推理 (US-only inference)：** 针对必须在美国境内运行的工作负载，以 1.1 倍的 Token 价格提供。

**产品端更新：**
*   **Claude Code 代理团队 (研究预览版)：** 您现在可以启动多个代理作为一个团队并行工作并自主协调，非常适合代码库审查等可拆分的重度读取任务。您可以使用 Shift+Up/Down 或 tmux 直接接管任何子代理。
*   **Claude in Excel：** 性能大幅提升，能够处理耗时更长的复杂任务。它可以在行动前进行规划，摄取非结构化数据并在无指导的情况下推断出正确的结构，还能一次性处理多步更改。
*   **Claude in PowerPoint (研究预览版)：** 结合 Excel，您可以先在 Excel 中处理和结构化数据，然后在 PowerPoint 中将其可视化。Claude 会读取您的布局、字体和幻灯片母版以保持品牌一致性。该功能现面向 Max、Team 和 Enterprise 计划开放。

### 安全性与网络防御

智能的提升并未以牺牲安全为代价。在我们的自动化行为审计中，Opus 4.6 在欺骗、阿谀奉承、鼓励用户妄想以及协助滥用等不良行为上的发生率极低。它与我们迄今为止对齐度最高的前沿模型 Opus 4.5 一样安全。同时，Opus 4.6 的“过度拒绝率”（即模型未能回答无害查询的情况）也是近期所有 Claude 模型中最低的。

我们为 Opus 4.6 运行了迄今为止最全面的安全评估，首次应用了许多新测试（如用户福祉评估），并升级了多项现有测试。我们还尝试了来自“可解释性（interpretability）”领域的新方法，以了解模型特定行为的内在原因，从而捕捉标准测试可能遗漏的问题。

鉴于该模型展现出增强的网络安全能力，我们开发了 6 种新的网络安全探针来追踪潜在的滥用形式。同时，我们也在加速模型的网络防御应用，利用它来帮助发现和修复开源软件中的漏洞。网络安全发展迅速，我们将在不久的将来实施实时干预以阻止滥用。

Claude Opus 4.6 现已在 claude.ai、我们的 API 以及所有主要云平台上线。常规 API 定价保持不变，为每百万 Token $5（输入）/ $25（输出）。
