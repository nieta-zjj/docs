# Claude Code Security：为防御者提供前沿网络安全能力

## 文档信息
- 来源：https://www.anthropic.com/news/claude-code-security

## 摘要
**1) 一句话总结**
Claude Code Security 是一项新推出的限量研究预览版功能，它利用 AI 深入推理和扫描代码库，以发现传统静态分析遗漏的复杂漏洞，并提供需人类审查和批准的修复补丁。

**2) 关键点**
*   **开放对象**：目前向企业（Enterprise）和团队（Team）客户开放限量研究预览版，开源项目维护者可申请免费的加急访问权限。
*   **工作原理**：突破传统基于规则的静态分析局限，像人类研究员一样理解组件交互和追踪数据流，能够捕获业务逻辑缺陷或访问控制失效等复杂漏洞。
*   **多阶段验证机制**：漏洞发现需经过自我验证与过滤（减少误报）、严重程度评级、置信度评估，最后提交给分析师。
*   **人类主导决策**：系统仅负责发现问题并提出补丁建议，所有修复必须在仪表板中经人类审查和批准，不会自动应用。
*   **实战成果**：借助 Claude Opus 4.6 模型，团队已在生产环境的开源代码库中发现了 500 多个潜伏多年的漏洞，目前正与维护者进行负责任的披露。
*   **研发与测试背景**：基于一年多的研究，经过前沿红队（Frontier Red Team）在 CTF 比赛及与太平洋西北国家实验室（PNNL）合作项目中的系统性压力测试。
*   **内部验证**：Anthropic 已在内部使用该工具审查自身代码，证实了其在保护系统方面的有效性。

**3) 风险/差距**
*   AI 帮助发现和修复漏洞的能力，同样可能被攻击者用来进行恶意的漏洞利用。
*   攻击者将利用 AI 比以往更快地发现代码中可利用的弱点。
*   部分安全问题包含仅从源代码难以评估的细微差别（因此系统需提供置信度评级并依赖人类最终决策）。

## 正文
网页版 Claude Code 的一项新功能——Claude Code Security，目前已推出限量研究预览版。它能够扫描代码库中的安全漏洞，并提出针对性的软件补丁供人类审查，从而帮助团队发现和修复传统方法通常会遗漏的安全问题。

安全团队普遍面临着一个挑战：软件漏洞太多，而处理这些漏洞的人手不足。现有的分析工具虽然有帮助，但作用有限，因为它们通常只能寻找已知的漏洞模式。要发现那些经常被攻击者利用的、微妙且依赖上下文的漏洞，需要经验丰富的人类研究员，但他们正面临着不断积压的工作。

AI 正在改变这一现状。我们最近展示了 Claude 能够检测出新型的高危漏洞。然而，帮助防御者发现和修复漏洞的能力，同样可能被攻击者用来进行漏洞利用。

Claude Code Security 的推出，正是为了将这种力量完全交到防御者手中，保护代码免受这类新型 AI 攻击。我们正向企业（Enterprise）和团队（Team）客户发布限量研究预览版，并为开源仓库的维护者提供加急访问权限，以便我们共同完善其功能并确保其被负责任地部署。

### Claude Code Security 的工作原理

静态分析是一种广泛部署的自动化安全测试形式，它通常是基于规则的，这意味着它会将代码与已知的漏洞模式进行匹配。这种方法能捕获常见问题（如暴露的密码或过时的加密），但往往会遗漏更复杂的漏洞（如业务逻辑缺陷或访问控制失效）。

Claude Code Security 不扫描已知模式，而是像人类安全研究员一样阅读和推理你的代码：理解组件之间如何交互、追踪数据在应用程序中的流动，并捕获基于规则的工具所遗漏的复杂漏洞。

每个发现都会经过多阶段的验证过程，然后才会提交给分析师：
*   **自我验证与过滤**：Claude 会重新检查每个结果，尝试证明或证伪自己的发现，从而过滤掉误报。
*   **严重程度评级**：为漏洞分配严重程度评级，以便团队可以优先处理最重要的修复工作。
*   **置信度评估**：由于这些问题通常包含仅从源代码难以评估的细微差别，Claude 还会为每个发现提供置信度评级。
*   **人类主导决策**：经过验证的发现会显示在 Claude Code Security 仪表板中，团队可以在此审查问题、检查建议的补丁并批准修复。没有任何修复会在未经人类批准的情况下被应用——Claude 负责发现问题并提出解决方案，但最终决定权始终在开发者手中。

### 研发背景与实际应用

Claude Code Security 建立在对 Claude 网络安全能力一年多的研究基础之上。我们的前沿红队（Frontier Red Team）对这些能力进行了系统性的压力测试：让 Claude 参加竞争性的 CTF（夺旗）比赛，与太平洋西北国家实验室（PNNL）合作探索使用 AI 保卫关键基础设施，并不断提升 Claude 发现和修补实际代码漏洞的能力。

因此，Claude 的网络防御能力得到了大幅提升。使用本月早些时候发布的 Claude Opus 4.6，我们的团队在生产环境的开源代码库中发现了 500 多个漏洞——尽管经过了多年的专家审查，这些漏洞仍潜伏了数十年未被发现。我们目前正与维护者一起进行漏洞分类和负责任的披露，并计划扩大我们与开源社区的安全合作。

我们也在内部使用 Claude 来审查自己的代码，并发现它在保护 Anthropic 系统方面非常有效。我们构建 Claude Code Security 就是为了让这种防御能力得到更广泛的应用。由于它构建在 Claude Code 之上，团队可以在他们已经使用的工具中审查发现并迭代修复方案。

### 行业前景与未来之路

网络安全正处于一个关键时期。鉴于 AI 模型在发现长期隐藏的错误和安全问题方面变得如此高效，我们预计在不久的将来，全球很大一部分代码都将被 AI 扫描。

攻击者将利用 AI 比以往更快地发现可利用的弱点。但行动迅速的防御者同样可以发现这些弱点并进行修补，从而降低攻击风险。Claude Code Security 是我们迈向更安全的代码库、提升全行业安全基准这一目标的重要一步。
